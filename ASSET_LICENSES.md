# Asset Licenses — Paila (पाइला)

> This file tracks the exact license, source, and attribution for every image
> bundled in the app. It is the single source of truth for asset compliance.
>
> **Rules:**
> 1. NEVER bundle an image without adding it to this file first
> 2. NEVER use an image with an unverified license
> 3. All CC BY / CC BY-SA images MUST have attribution visible in the app (Settings > Acknowledgements)
> 4. Prefer Unsplash / Pexels / Pixabay / NASA (no attribution required) over Wikimedia CC BY-SA
> 5. If an image license cannot be verified, DO NOT USE IT

## App Store / Play Store Compliance Notes

### Apple App Store
- All images must be legally licensed for commercial distribution
- If using CC BY-SA content, attribution must be accessible in-app
- Screenshots in App Store listing must use images you have rights to
- Apple does not accept "fair use" as a defense for unlicensed imagery

### Google Play Store
- Same commercial license requirements as Apple
- Data Safety section: images are bundled (not collected from users)
- Content rating: nature photography is unrated content (no issue)

### In-App Attribution
- An "Acknowledgements" or "Photo Credits" section must be accessible from Settings
- Format: "Photo: [Description] by [Photographer] via [Source] ([License])"
- For Unsplash/Pexels/Pixabay: attribution not legally required but good practice
- For CC BY / CC BY-SA: attribution IS legally required

## How to Add a New Image

1. Find image on Unsplash/Pexels/Pixabay/NASA (preferred) or Wikimedia Commons
2. Verify the license on the actual image page (not the search results page)
3. Download the image
4. Resize to max 1200px wide, JPEG ~80% quality
5. Save to `assets/images/milestones/{slug}.jpg`
6. Add an entry to the "Currently Bundled Assets" section above with ALL fields filled
7. Update `src/shared/assets/milestone-images.ts` to add the `require()` call
8. If CC BY or CC BY-SA: add attribution to the in-app acknowledgements data
